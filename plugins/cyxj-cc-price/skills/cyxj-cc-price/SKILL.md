# CC价格计算 — Claude 订阅成本计算器

## When to Use

- 用户说 `/CC价格计算`、"算一下拼车"、"中转多少钱合理"
- 用户提供套餐类型 + 人数 + 价格，想知道值不值
- 用户想对比不同拼车方案

## 输入格式

支持灵活输入，例如：
- `/CC价格计算 20× 三人拼 ¥600`
- `/CC价格计算 5× 双人拼 ¥400`
- `/CC价格计算 中转 0.45元/刀 缓存80%`

## 内置数据表

### Credits 限额（官方数据）

```
套餐       | 5h窗口Credits  | 周Credits     | 5h窗口($) | 日($)  | 周($)    | 月($)
Pro $20    | 550,000       | 5,000,000    | 4.1      | 5.36  | 37.50   | 162.50
5× $100    | 3,300,000     | 41,666,700   | 24.8     | 44.64 | 312.50  | 1,354.17
20× $200   | 11,000,000    | 83,333,300   | 82.5     | 89.29 | 625.00  | 2,708.33
```

### 换算常量

```
1M Credits = $7.5
月系数 = 4.333 周/月
日 = 周限 ÷ 7
默认汇率 = 7.2 元/刀
默认缓存命中率 = 82.9%
缓存加成系数 = 1.558（基于 82.9% 命中率）
```

### 缓存加成公式

```
缓存加成系数 = 1 / (1 - cache_ratio × 0.675)
  其中 0.675 ≈ cache_read_price / input_price 的节省比例
  82.9% 命中率 → 系数 ≈ 1.558
```

## 计算逻辑

### 场景 A：拼车计算

输入：套餐类型、拼车人数、月费（人民币）

```
1. 查表获取套餐的 周限额($)
2. 人均周限 = 周限额 ÷ 人数
3. 人均日限 = 人均周限 ÷ 7
4. 人均月限 = 人均周限 × 4.333
5. 元/刀(无缓存) = 月费 ÷ 人均月限 ÷ 汇率
6. 元/刀(含缓存) = 月费 ÷ (人均月限 × 缓存加成系数) ÷ 汇率
```

### 场景 B：中转定价评估

输入：中转价格（元/刀）、可选缓存命中率

```
1. 对比基准线：
   - 0.53 元/刀 = 无缓存订阅成本
   - 0.34 元/刀 = 含缓存(82.9%)真实成本
2. 计算性价比 = 0.53 ÷ 中转价格
3. 如提供缓存率，重新计算加成系数
```
